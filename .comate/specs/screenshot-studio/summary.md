# Hijessy 跨平台截图工具 — 完成总结

## 交付概览

按里程碑增量实现了一款 **跨平台、轻量、高性能** 的 Rust 截图工具，10 个任务全部完成，`cargo build --release` 通过，产物 **6.9 MB**（macOS，已启用 `opt-level="z" + lto + strip + panic=abort`）。

## 技术栈

| 层 | 选型 |
| --- | --- |
| UI | eframe/egui 0.35（即时模式 GPU 渲染） |
| 屏幕捕获 | xcap 0.9.6（全屏/窗口，未来录屏同库） |
| 剪贴板 | arboard 3.6 |
| 全局快捷键 | global-hotkey 0.8 |
| 图像/编码 | image 0.25 |
| 文字栅格化 | ab_glyph（复用系统 CJK 字体，不内置字体） |
| 配置 | serde + toml |

> 说明：标注合成刻意采用手写像素绘制而非引入 imageproc（其依赖 nalgebra/rustfft 过重），以保障"轻量"目标。

## 已实现功能

- **捕获**：全屏、窗口、鼠标框选（冻结屏 + 全屏遮罩 + 拖拽选区，Enter/双击确认、Esc 取消）、长截图（滚动逐帧抓取 + 重叠条带匹配拼接，含回退堆叠）。
- **标注**：矩形、椭圆、箭头（细线/实心两种样式）、文字（小14/中20/大28 三档字号）、序号（点击自增、圆形数字标记）、马赛克；8 色板、线宽（细/中/粗）；撤销/重做（含 Ctrl+Z / Ctrl+Y）。
- **顶部工具栏**：工具组 + 属性组（随工具动态显示颜色/线宽/字号/箭头样式）+ 操作组（💾 保存 / ✕ 取消 / ✓ 确认）。
- **输出**：确认默认写入剪贴板；保存按钮落盘为 PNG/JPEG（时间戳命名，可配置目录/格式）。
- **快捷键**：全局热键 框选 `CmdOrCtrl+Shift+A` / 全屏 `+F` / 窗口 `+W`，可经配置文件覆盖；注册失败自动降级为界面触发。
- **配置**：首次运行写出 `~/.config/hijessy/config.toml` 默认模板。

## 架构与扩展点

单 crate 分层模块，以 trait 隔离"捕获 / 编辑 / 输出"三层管线：

```
capture (CaptureEngine / CaptureSource / Frame)   ← 单帧=截图；未来 FrameStream=动图/录屏
editor  (Annotation 模型 + compose 合成器 + 画布)
output  (OutputSink / Artifact)                    ← 现 Image；未来 Gif / Video
selection / hotkey / config / fonts
```

未来加法式扩展（不改既有代码）：
- **GIF/录屏**：新增 `FrameStream` 与 `Artifact::Gif/Video`，复用 xcap 录制能力与现有输出层。
- **OCR**：新增 `Recognizer` trait 消费选区图像，以可选 feature + 模型按需下载方式接入。

## 已知限制

- **Linux Wayland**：xcap 对 Wayland 支持受限，完整支持为 X11 / macOS / Windows；Wayland 后续经 xdg-desktop-portal 补充。
- **框选/长截图与自身窗口**：主窗口若覆盖目标区域可能被截入；长截图为手动逐帧的 best-effort 版本，需将窗口移开滚动区域。
- **多显示器**：当前框选覆盖主显示器；多屏统一遮罩后续完善（`list_monitors` 已预留）。
- **文字/序号 CJK**：依赖系统 CJK 字体；未找到时中文可能无法正确显示/烧录（已按平台列出常见字体路径探测）。
- **macOS**：首次截屏需在系统设置授予"屏幕录制"权限。

# 迭代二总结：Snipaste 风格一体化浮层 + 美化

针对"太丑、交互绕"的反馈，将"独立主窗口 + 分步截图/编辑"重构为**单一全屏浮层的一体化体验**，6 个任务全部完成，`cargo check` 无告警、`cargo build --release` 通过、二进制 **6.9 MB**。

## 主要变化

- **启动即截图、无主界面**：进程以全屏透明、鼠标穿透、置顶窗口常驻；启动/快捷键触发时关闭穿透、冻结全屏进入截图浮层；确认/取消后回到穿透空闲态（界面不可见、不拦截桌面操作），等待下次唤起。
- **默认框选当前窗口**：进入浮层后自动高亮**光标下窗口**（多窗口命中取最小面积者）为默认选区；移动光标实时切换。
- **模式切换**：选区右上外侧「全屏 / 窗口」图标按钮（Tooltip）；任意处拖拽即切换为自定义框选。
- **选区手柄**：8 个圆点手柄可二次拖拽调整（移动 / 缩放），左上角显示 `宽×高` 尺寸标签，选区外压暗。
- **底部胶囊工具栏**：选区确定后于下方浮现圆角胶囊，**纯矢量图标按钮 + 悬浮 Tooltip**；工具（选择/矩形/椭圆/箭头/文字/序号/马赛克）+ 样式弹层（8 色 / 线宽 / 字号 / 箭头样式）+ 撤销/重做 + 保存/取消/确认。
- **Light 主题**：`egui::Visuals::light()`，蓝色高亮 `#2B7FFF`，胶囊白底圆角阴影。
- **图标方案**：放弃 egui-phosphor（其 0.12 依赖 egui 0.34，与本项目 egui 0.35 冲突），改为 `overlay/icons.rs` 手写矢量线性图标，零额外依赖、契合"轻量"。
- **输出**：标注以冻结全屏像素坐标存储，`compose` 合成后按选区裁剪 → 确认复制剪贴板 / 保存落盘。

## 结构变化

- 新增 `overlay/{mod.rs, icons.rs}`（承载窗口识别、模式切换、选区手柄、遮罩、标注、胶囊工具栏）。
- `editor/` 精简为 `model` + `compose`（合成器复用为输出烧录）。
- 移除旧 `selection/` 与独立 `Editor` 画布/主窗口按钮。
- 长截图模块 `capture/scroll.rs` 暂未接入新浮层入口，保留（已测试）待后续里程碑接入。

## 验证

- 交互（浮层显示、窗口高亮、拖拽/手柄、图标工具栏、标注、复制/保存）需在桌面会话手动实测；建议 `cargo run --release` 体验，Esc 取消。

## 已知限制（迭代二新增/相关）

- **常驻窗口**：空闲态为全屏透明穿透窗口，无系统托盘；退出需 Cmd+Q / 结束进程（后续可加托盘）。
- **透明捕获时序**：截图前依赖空闲态透明帧被合成为桌面；极端时序下可能需一帧稳定，实测如有偏差再微调（可改为捕获前强制一帧清屏）。
- **窗口识别**：基于 xcap 窗口几何 + 最小面积命中；Wayland/跨屏窗口可能不准，退化为手动框选 + 全屏。
- **马赛克预览**：浮层内以磨砂占位显示，真实像素化在输出时烧录（所见与最终略有差异）。
